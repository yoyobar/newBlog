### HTML의 정의

> HTML`Hyper Text MarkUp Language`은 웹 페이지와 그 내용을 구조화하기 위해  사용하는 마크업(Mark-up) 언어입니다.


```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <meta http-equiv="x-ua-compatible" content="IE=Edge">
  <title>Document</title>
</head>
<body>
</body>
</html>
```

개행 테스트

`<!DOCTYPE html>{:html}`

가장 최신 웹 표준을 지원하는 HTML로 설정한다는 의미.

`<html lang="ko">{:html}`

html의 언어 기본설정을 한국어로 지정하는 내용이다. 기본 값인 `en`을 사용할 경우
불필요하게 사용자의 크롬 등 브라우저에게 번역 요청을 보내는 경우도 있으므로 확인할 것.

`<meta charset=”UTF-8”>{:html}`

html 파일의 인코딩을 알려주는 태그이다. 컴퓨터가 HTML 파일의 정보 형태가 어떤 언어로 되어 있는지에 따라 선택지가 달라지나, 일반적인 경우 `UTF-8`을 사용한다는 점만 알아두자.

`<meta http-equiv=”x-ua-compatible” content=”IE=Edge”>{:html}`

`content=”IE=Edge"`의 경우 IE(Internet Explorer)의 최신 표준 모드를 선택하는 문서 모드다. 현 시대에서 일반적으로 IE는 이제 사용되지 않지만, 호환성을 위해 주로 사용한다.

`http-equiv=”x-ua-compatible”` 은 간단하게 설명하자면 지정한 버전에 맞게 표준모드로 제한 한다는 의미다. 즉 IE=Edge 표준에 따라 기능을 제한하는 의미로 가볍게 알고 있으면 될 것 같다.

### 링크 표시하기

링크는 `<a>` 태그로 표시한다. 해당 태그에는 부가적인 태그가 존재한다. 

| target |  `_blank`  새 탭에서 페이지 열기 | `_target` 페이지 열기 |
| --- | --- | --- |

`href` 는 원하는 링크를 입력하면 된다. 아래의 예제를 참고하자.

```html
...
<body>
	<a href="https://github.com/yoyobar" target="_blank">새탭에서 페이지 열기</a>
	<a href="https://github.com/yoyobar" target="_self">페이지 열기</a>
</body>
</html>
```

<Preview>
	<a className="text-indigo-500" href="https://github.com/yoyobar" target="_blank">새탭에서 페이지 열기</a>
	<a className="text-indigo-500" href="https://github.com/yoyobar" target="_self">페이지 열기</a>
</Preview>


여기서 생성하는 `href` 는 다른 인터넷 링크뿐만아니라, 기존 프로젝트내에서의 이동도 가능하다.
즉 거대한 서비스를 만들고자 한다면 다양한 페이지로 구성해야 될텐데, 그를 위한 초석이라고 생각하면 간단하다.

### 목록 만들기

목록은 `<ol>`  `<ul>` 로 구성되며, 내부 데이터는 `<li>` 로 구성된다.

`<ol>` 은 Order list의 줄임말로, 순차가 존재하는 데이터를 의미한다.

`<ul>` 은 Unordered list의 줄임말로, 순차가 존재하지 않는 데이터를 의미한다.

각각의 예시를 확인해보자

```html
##Orderlist를 인용한 예시##
...
<body>
	<ol>
		<li> 복숭아 </li>
		<li> 사과 </li>
		<li> 바나나 </li>
	</ol>
</body>
</html>
```

<Preview>
	<ol>
		<li> 복숭아 </li>
		<li> 사과 </li>
		<li> 바나나 </li>
	</ol>
</Preview>

위에 있는 `<ol>`의 코드는 위와 같은 값으로 나오게 된다. 데이터가 순차적으로 존재하는 것이다. 추가적으로 `start` , `type` 속성을 인용하여 시작 번호와, 번호의 종류를 선택할 수 있다.

```html
##Unorderlist를 인용한 예시##
...
<body>
	<ul>
		<li> 복숭아 </li>
		<li> 사과 </li>
		<li> 바나나 </li>
	</ul>
</body>
</html>
```

<Preview>
	<ul>
		<li> 복숭아 </li>
		<li> 사과 </li>
		<li> 바나나 </li>
	</ul>
</Preview>

`<ul>` 의 코드는 위에 있는 값과 같이 별도의 숫자가 매겨져 있지 않는 데이터로 나타나게 된다.

### 표 만들기

표는 열과 행의 대한 개념을 확실히 알고 가는게 좋다.


열(`rows`)은 **세로**로 세며 **가로 값**을 뜻한다. 줄이라는 의미다.

행(`cols`)은 **가로**로 세며 **세로 값**을 뜻한다. columns의 약자며, 기둥을 뜻한다.

표는 구성하는 요소가 많다.

| `<table>` | 표 생성 태그 |
| --- | --- |
| `<tr>` | 행 생성 태그 |
| `<th>` | 제목 생성 태그 |
| `<td>` | 값 생성 태그 |
| `<caption>` | 타이틀 제목 태그 |
| `<rowspan>` | 가로 병합 |
| `<colspan>` | 세로 병합 |

행, **세로 값**을 늘리고자 한다면 `<tr>` 을 그만큼 늘리면 되는 것이다.

열, **가로 값**을 늘리고자 한다면 `<tr>` 에 원하는 만큼 항목을 생성하면 되는 것이다.

또한, `<tr>` 은 `<thead>, <tbody>, <tfoot>` 로도 표기할 수 있는데, 이럴 경우 표를 직관적으로 이해하는데 더 도움을 줄 수 있다. 그리고 `<table>` 속성에 `border` 값을 주지 않을경우, 표가 더러워 보이니 외곽선을 잘 넣어주자

아래 예시를 확인해보자

```html
...
<body>
  <table border="1">
    <caption>
      오늘의 할일
    </caption>
    <tr>
      <th>제목</th>
      <th>해야할 일</th>
      <th>완료 여부</th>
    </tr>
    <tr>
      <td>중요한 일</td>
      <td>잠자기</td>
      <td>X</td>
    </tr>
  </table>
</body>
</html>
```


이러한 표로 완성되는 모습을 확인 할 수 있다. 상기 코드에서 `<tr>` 은 `<thead> <tbody> <tfoot>` 등 상황에 맞게 직관적인 태그를 달 수 있으니 참고할 것

`rowspan`과 `colspan` 으로 셀의 가로, 세로 병합도 가능하니 참고하자.

### Input 태그 및 입력 태그

사용자에게 입력을 받는 태그로 대표적인 html태그는 `<input>` 태그다. html이 발전함에 따라 다양한 종류가 늘어나기도 했지만, 각각 사용 방법이 조금씩 다르니 기록해보자.

```html
<input id="id" type="text" placeholder="아이디를 입력하세요." />
<label for="id">아이디</label>
```
<Preview>
<input placeholder="아이디를 입력하세요." className="bg-slate-200 rounded-md p-2"/>
</Preview>

`type="text"` 의 경우 간단하게 입력하는 창이 나온다.  

입력하는 창에 `placeholder` 를 통해 사용자가 직관적으로 알도록 한다.

`label`은 `for` 을 통해 각 데이터에 맞는 값을 입력한다.

**기본적인 Input의 종류**

| type  | 용도 | 특이 사항 |
| --- | --- | --- |
| `text` | `text` 타입 입력 | `placeholder` 사용가능 |
| `password` | `password` 타입 입력, `blind` | `placeholder` 사용가능 |
| `email` | email 타입 입력 | `placeholder` 사용가능 |
| `range` | 조정 할 수 있는 `bar`를 생성 |  |
| `checkbox` | 체크 할 수 있는 `box`를 생성 | 다중 선택이 가능, name 태그로 서로 연동해두는게 중요 |
| `radio` | 체크 할 수 있는 `radio`를 생성 | 다중 선택이 불가, name 태그로 서로 연동해두는게 중요 |

<Preview>
<div className="flex-col flex gap-1 items-start">
	<input type="text" placeholder="text" className="bg-slate-200 rounded-md p-2 w-1/3"/>
	<input type="password" placeholder="password" className="bg-slate-200 rounded-md p-2 w-1/3"/>
	<input type="email" placeholder="email" className="bg-slate-200 rounded-md p-2 w-1/3"/>
	<input type="range" className="bg-slate-200 rounded-md p-2"/>
	<input type="checkbox" className="bg-slate-200 rounded-md p-2"/>
	<input type="radio" className="bg-slate-200 rounded-md p-2"/>
</div>
</Preview>

**그외 입력태그**

| 태그 명 | 용도 | 특이 사항 |
| --- | --- | --- |
| `Select`  | 다중 확인이 가능한 목록 생성 | 자식 태그로 `Option` 를 사용하여 데이터를 늘림
각 데이터 별로 value 를 사용하여 반환 값 설정 가능 |
| `fieldset`  | 입력폼을 묶는 상자 생성 | 현재 html에서는 주로 사용되진 않음, 
legend 를 통해 범례 작성 가능 |
| `textarea`  | 자유롭게 입력할 수 있는 텍스트 box 생성 | 우측 끝을 잡아 사이즈를 수동 조정할 수 있음 |
| `progress`  | 진행도를 나타내는 bar 생성 | max 값을 설정하고, value 로 현재 값을 나타낸다. |

<Preview>
<div className="flex-col flex gap-1 items-start">
	<select className="bg-slate-200">
	<option>선택 1</option>
	<option>선택 2</option>
	<option>선택 3</option>
	</select>
	<fieldset className="flex flex-col gap-1 items-start w-full">
	<input type="text" placeholder="text" className="bg-slate-200 rounded-md p-2 w-1/3"/>
	<input type="password" placeholder="password" className="bg-slate-200 rounded-md p-2 w-1/3"/>
	</fieldset>
	<textarea className="bg-slate-200 rounded-md"></textarea>
	<progress max="100" value="25"/>
</div>
</Preview>

### 시맨틱 태그

HTML이 업데이트 됨에 따라, 협업에 용이하게 각각의 태그의 이름을 구별하기 쉽게 기능적으로 추가되었다. 직접 HTML을 작성할 때 이용한다면 HTML 작성 능력에 도움이 될 것이라고 판단된다.

```html
<!-- 독립적인 컨텐츠 -->
<article></article>
<!-- 별도의 컨텐츠 -->
<aside></aside>
<!-- 추가 상세정보 -->
<details></details>
<!-- figure의 자막 -->
<figcaption></figcaption>
<!-- 설명이 붙는 독립 컨텐츠 -->
<figure></figure>
<!-- 맺음 말 -->
<footer></footer>
<!-- 머릿 말 -->
<header></header>
<!-- 주된 콘텐츠 -->
<main></main>
<!-- 네비게이션 링크 -->
<nav></nav>
<!-- 콘텐츠의 한 섹션 -->
<section></section>
<!-- detail의 헤더 -->
<summary></summary>
<!-- 강조할 시간 -->
<time></time>
```
